# 人狼オフライン

友達とオフラインで遊ぶための、ブラウザだけで動く人狼進行ツールです。
サーバー不要で、フォルダをそのまま配って使えます。

## 主な機能

- プレイヤー登録、順番並べ替え（↑ / ↓ / 削除）
- 役職人数の可変設定（市民は自動調整）
- 役職公開フェーズ（1人ずつ端末を回して確認）
- 夜アクション（本人確認 → アクション開始 → 完了）
- 朝フェーズ（犠牲者・各種アナウンス）
- 投票フェーズ（本人確認 → 対象選択 → 最終確認）
- 同票処理（再投票 or ランダム処刑）
- 勝利画面（陣営勝利を大きく表示、勝者一覧・全役職・投票履歴）
- PWA（ホーム画面追加・オフライン起動）

## 主要ルール

### 勝利条件

- 生存人狼（`人狼 + 能ある人狼`）が0人になったら市民側勝利
- 生存人狼数が生存市民陣営人数以上になったら人狼側勝利
- 全員死亡した場合は引き分け

### 占い・霊媒の判定表示

- 黒判定: `人狼`
- 白判定: `人狼ではない`
- 表示文: `Aは人狼。` / `Bは人狼ではない。`

### 初日（1日目）占い設定

- `ランダム白`: 占い結果を自動表示（`人狼 / 能ある人狼 / サイコ` は候補外）
- `あり`: 対象を手動選択
- `なし`: 初日占いなし

### 騎士設定

- 連続ガード `あり / なし`
- `なし` の場合、前夜と同じ対象は護衛不可

### 人狼設定

- 人狼の投票・襲撃先表示 `見える / 見えない`
- `見える` の場合、投票時・夜襲撃時に他の人狼系の投票先を表示

### 同票設定

- `再投票`: 1回だけ再投票、再同票ならランダム処刑
- `ランダム処刑`: 同票時は即ランダム

## 役職ごとの特殊挙動

### 能ある人狼

- 人狼としてカウントされる（勝利判定・占い霊媒黒判定の対象）
- 夜アクションで `襲撃しない` を選べる
- 人狼系の襲撃票が全て `襲撃しない` なら襲撃は発生しない

### 狂信者 / ささやく狂人 の見え方

- 狂信者・ささやく狂人には、`人狼 + 能ある人狼` を `人狼` として表示
- ささやく狂人が1人以上いる場合のみ、人狼側に `ささやく狂人` 行を表示

### サイコ

- 占い・霊媒は白判定（`人狼ではない`）
- 襲撃では死亡しない
- サイコに関わった行動者は翌朝死亡
- サイコが襲撃対象になった場合、人狼系の中からランダム1人が死亡

## フェーズ進行

1. プレイヤー追加と配役設定
2. 役職確認（順番に確認）
3. 1日目夜（初日アクション）
4. 朝フェーズ
5. 投票フェーズ
6. 夜フェーズ
7. 以降は朝→投票→夜を繰り返し

## iPhone / Android で使う

### Android

- フォルダごとスマホにコピー
- ファイル管理アプリから `index.html` を開く

### iPhone（PWA推奨）

iOS は `file://` の制限があるため、初回はローカル配信経由が安定です。

1. PCでローカル配信
2. iPhone Safariでアクセス
3. 共有 → ホーム画面に追加
4. 以後はオフライン起動可能

```bash
cd '/Users/ababa/Library/Mobile Documents/com~apple~CloudDocs/人狼'
python3 -m http.server 8080
```

```bash
ipconfig getifaddr en0
```

IP が `192.168.1.23` なら:

```text
http://192.168.1.23:8080
```

## 役職追加しやすい構成

配役UI・配役計算は `ROLE_DEFINITIONS` と `ROLE_ORDER` を中心に自動化しています。

- カウンターUIは `ROLE_DEFINITIONS` から自動描画
- 役職名・説明は `ROLE_DEFINITIONS` を参照
- バリデーションとデッキ生成は `ROLE_ORDER` / `ADJUSTABLE_ROLES` で処理

### 新役職追加の最小手順

1. `app.js` の `ROLE_DEFINITIONS` に追加
2. `ROLE_ORDER` にキーを追加
3. 必要なら夜アクション・判定ロジックを追加
