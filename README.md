# 人狼オフライン

友達とオフラインで遊ぶための、ブラウザだけで動く人狼配役ツールです。
サーバー不要で、フォルダをそのまま配って使えます。

## できること

- プレイヤー名の追加と配役設定を同一画面で実行
- プレイヤー順の並べ替え（↑ / ↓）
- 配役設定（人狼 / 市民 / 占い師 / 霊媒師 / **騎士 / 狂人** の人数指定）
- 配役人数を `＋ / −` で調整（市民は自動調整）
- 端末を回して順番に秘密確認
- 朝 → 夜 → 投票 → 勝利判定のサイクル進行
- 同票時の処理（再投票 / ランダム処刑）
- フェーズ共通タイマー
- **生存者と犠牲者の両方を表示（役職は非表示）**
- **投票フェーズで対象確認機能（「◇◇さんですか？」と確認してから投票）**

## 使い方（PC / スマホ共通）

1. このフォルダを開く
2. `index.html` をブラウザで開く
3. プレイヤー名を追加し、必要に応じて順番を並べ替える
4. 同じ画面で `＋ / −` で人狼・占い師・霊媒師・**騎士・狂人** を調整（市民は自動計算）し、「配役を確定」
5. 1人ずつ「役職を見る」→「確認して次へ」
6. 全員確認後、`0日目夜` を開始（1番のプレイヤーから順番にアクション）
7. 0日目夜完了後、1日目朝へ進行
8. 以降は 朝 → 投票 → 夜 → 朝... を繰り返し、勝利条件を満たしたら終了

## 配役制約（実装済み）

- 人狼は過半数以上に設定不可
- 市民人数は `全体人数 - (人狼 + 占い師 + 霊媒師)` で自動算出
- 市民人数がマイナスになる設定時は、市民表示を赤で強調

## 0日目夜の占い師アクション

- 0日目は全員が順番にアクションを実施
- 占い師の番では、`人狼以外` のランダム1名が自動で `白` と表示

## ゲーム進行ルール（実装済み）

- 夜フェーズ
	- 生存者全員が順番に夜アクションを実行
	- 人狼は襲撃対象を投票（複数人狼で同票ならランダム決定）
	- 占い師は対象を指定して `黒 / 白` を確認（人狼のみ黒）
	- 霊媒師は当日朝の投票犠牲者を `黒 / 白` で確認（人狼のみ黒）
	- 騎士は自分以外1人を護衛（襲撃対象と一致した場合は死亡回避）
	- 市民・狂人はダミーアクション
- 朝フェーズ
	- 議論用フェーズ（タイマー利用可）
- **投票フェーズ**
	- **生存者が順番に投票対象を選択 → 「◇◇さんですか？」確認表示 → 「投票する」ボタンで確定**
	- **デフォルトでは自分投票不可（切替可）**
	- **同票時は再投票またはランダム処刑を選択可能**
- **生存者・犠牲者表示**
	- **ゲーム進行中、生存者と犠牲者を両方表示（役職は表示しない）**
- **ゲーム終了画面**
	- **全員の役職を開示（生存 / 犠牲の別）**
	- **全投票の投票元 → 投票先を表示**
- 勝利判定
	- 人狼が全滅したら市民勝利
	- 生存人狼数が生存市民数以上なら人狼勝利

## UIの工夫

- **重要情報の強調**
	- **勝者表示**: 大きく、目立つ色（緑）で表示
	- **現在の投票者名**: アクセント色で大きく表示
	- **昨夜の犠牲者**: 赤色で目立つ
	- **フェーズ名**: 青色でボーダー付き
	- **現在のラウンド・フェーズ**: バッジで見やすく表示
- **プレイ中の表示**
	- ゲーム中は、プレイヤーのid・役職は表示しない
	- 生存者・犠牲者リストのみ名前を表示

### Android の場合

- フォルダごとスマホにコピー
- ファイル管理アプリから `index.html` を開く

### iPhone の場合

iOS は `file://` の扱いが制限されるため、`index.html` の直接起動は不安定です。

## iPhone単体でアプリのように使う（PWA）

1. 最初の1回だけ、同一Wi-Fi上のPCからローカル配信して開く
2. iPhoneのSafariでページを開く
3. 共有ボタン → 「ホーム画面に追加」
4. 以後はホーム画面アイコンから起動（スタンドアロン表示）

cd '/Users/ababa/Library/Mobile Documents/com~apple~CloudDocs/人狼'

## 役職追加をしやすくした構成

現在は `app.js` の `ROLE_DEFINITIONS` と `ROLE_ORDER` を中心に、配役UIと配役計算を自動生成します。

- 配役設定の `＋ / −` カウンターUIは `ROLE_DEFINITIONS` から自動描画
- 役職名・役職説明（公開時）は `ROLE_DEFINITIONS` から参照
- 配役バリデーションと役職デッキ生成も `ROLE_ORDER` / `ADJUSTABLE_ROLES` で処理

### 新しい役職を増やす最小手順

1. `app.js` の `ROLE_DEFINITIONS` に役職を追加
	- `label`, `camp`, `adjustable`, `defaultCount`, `setupHelp`, `revealDescription`
	- 双子のような特殊カウントなら `pairOnly: true`
2. `ROLE_ORDER` に役職キーを追加
3. その役職固有の能力がある場合のみ、夜アクション/判定ロジックを追加

※ これにより、単純な配役追加なら `index.html` の手編集は不要です。
